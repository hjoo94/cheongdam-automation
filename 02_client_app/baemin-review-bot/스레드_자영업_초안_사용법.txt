스레드 자영업 글 초안 기능 사용법

1. 이 기능이 하는 일

이 기능은 최근 일주일 기준으로 자영업 관련 키워드를 참고해서 스레드에 올릴 글 초안을 만들어줍니다.
목표는 다른 사람의 글을 복사하는 것이 아니라, 반응이 좋아 보이는 글의 분위기와 고민 포인트만 참고해서 우리 매장에 맞는 새 글을 만드는 것입니다.

초기버전에서는 안정성을 위해 자동 게시를 하지 않습니다.
프로그램은 초안까지만 만들고, 실제 스레드 게시 여부는 사용자가 직접 확인한 뒤 결정해야 합니다.

2. 기본 사용 순서

1) 프로그램을 실행합니다.
2) 상단 메뉴에서 "스레드 초안"으로 이동합니다.
3) "검색 키워드"에 참고하고 싶은 키워드를 입력합니다.
   예시:
   자영업자이야기, 자영업, 소상공인, 장사, 매출, 배민, 쿠팡이츠
4) "초안 방향"에 원하는 글 분위기를 적습니다.
   예시:
   사장님끼리 공감되는 현실적인 글, 댓글을 부르는 질문형, 특정 메뉴 홍보는 약하게
5) Threads Access Token이 있으면 입력합니다.
   토큰이 없으면 비워도 됩니다.
6) API 결과가 부족하거나 토큰이 없으면 "참고 글 직접 붙여넣기"에 스레드 글을 붙여넣습니다.
   글과 글 사이는 빈 줄로 구분합니다.
7) "스레드 초안 생성" 버튼을 누릅니다.
8) 오른쪽 "초안" 영역에서 마음에 드는 글을 확인합니다.
9) 문장, 매장 상황, 날짜, 메뉴명, 표현을 직접 확인한 뒤 스레드에 게시합니다.

3. Threads Access Token이 없는 경우

토큰이 없어도 기능을 사용할 수 있습니다.
이 경우 프로그램이 스레드 API를 직접 검색하지 못하므로, 사용자가 참고 글을 직접 붙여넣어야 합니다.

추천 방식:
1) 스레드 앱이나 웹에서 "자영업자이야기", "자영업", "소상공인", "장사" 등을 검색합니다.
2) 반응이 좋아 보이는 글을 5개에서 10개 정도 찾습니다.
3) 글 본문을 복사해 "참고 글 직접 붙여넣기"에 넣습니다.
4) 각 글 사이는 빈 줄로 띄웁니다.

4. Threads Access Token을 쓰는 경우

Meta 개발자 앱과 Threads API 권한이 필요합니다.
권한이 없으면 검색 API가 실패할 수 있습니다.

자주 생기는 오류:
- 토큰 만료: 새 토큰을 발급해야 합니다.
- 권한 부족: Meta 앱 권한 또는 앱 리뷰가 필요할 수 있습니다.
- 빈 결과: 키워드가 너무 좁거나 API가 공개 글 조회를 제한할 수 있습니다.
- 조회수 없음: 공개 글의 조회수는 API 권한에 따라 제공되지 않을 수 있습니다.

초기버전에서는 API 오류가 나도 프로그램이 멈추지 않게 만들었습니다.
오류가 생기면 직접 붙여넣은 참고 글을 사용해서 초안을 생성합니다.

6. 좋은 참고 글 고르는 기준

반응이 높은 글이 항상 좋은 글은 아닙니다.
자영업자 계정에는 아래 기준이 더 중요합니다.

- 댓글이 많이 달린 글
- 사장님들이 공감할 만한 현실적인 고민이 있는 글
- 너무 공격적이지 않은 글
- 특정 사람이나 업체를 저격하지 않는 글
- 내 매장 상황에 맞게 바꿔 쓸 수 있는 글

7. 반드시 지켜야 할 점

다른 사람의 글을 그대로 복사하면 안 됩니다.
문장 구조, 표현, 사례, 농담, 고유한 경험을 그대로 따라 쓰면 표절처럼 보일 수 있습니다.

초안은 반드시 사람이 확인해야 합니다.
특히 아래 내용은 게시 전에 지우거나 바꿔야 합니다.

- 다른 사람 이름
- 매장명
- 지역명
- 특정 업체 비난
- 법적 분쟁처럼 보이는 표현
- 실제와 다른 매출, 주문수, 후기
- 너무 자극적인 표현

8. 추천 운영 방식

처음에는 하루 1개만 올리는 것을 추천합니다.
반응을 보고 다음 글의 방향을 조정하는 방식이 안정적입니다.

추천 루틴:
1) 오전에 키워드 검색
2) 반응 좋은 글 5개 확인
3) 초안 3개 생성
4) 가장 자연스러운 글 1개만 수정해서 게시
5) 다음날 댓글, 좋아요, 조회 흐름 확인

9. 자동 게시를 아직 넣지 않은 이유

초기버전에서는 안정성과 계정 안전이 더 중요합니다.
자동 게시를 바로 넣으면 부자연스러운 글이 올라가거나, API 권한 오류로 실패할 수 있습니다.

먼저 초안 생성과 수동 게시 흐름을 안정화한 뒤,
나중에 원하면 "초안 저장", "예약 게시", "게시 완료 기록" 순서로 확장하는 것이 좋습니다.
