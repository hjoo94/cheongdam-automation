3. 고객 앱 설계
============================================================

위치:
02_client_app\baemin-review-bot

기술:
- Electron 28
- Playwright 1.44
- Node.js CommonJS
- electron-builder NSIS installer

주요 엔트리:
- app\main.js
  Electron main process. 라이센스 저장/검증, 설정 저장, 자동 업데이트, 실행 프로세스 관리, 파일 선택, 재무 분석, 스레드 분석, 오류 요약 IPC를 처리한다.

- app\preload.js
  renderer에서 사용할 API를 contextBridge로 노출한다.
  주요 채널: run-bot, stop-bot, check-app-update, pick-finance-file, analyze-finance-file, analyze-threads-marketing, summarize-error-logs, save-license, load-license, verify-license.

- app\index.html
  고객 앱 화면이다. 설정 입력, 기능 선택, 실행 로그, 재무 분석, 스레드 초안 등의 UI가 들어 있다.

- index.js
  CLI/child process 방식으로 실제 플랫폼 자동화를 실행한다.
  실행 대상: runBaemin, runBaeminAnswered, runCoupangEats, runCoupangAnswered.

- gptClient.js
  고객 앱에서 GPT 관련 요청을 서버로 보내는 중계 클라이언트다.
  runtime.json에서 서버 주소, 라이센스 키, 매장명, 기능 권한, 답글 모드를 읽고 `/api/gpt/*` 서버 API를 호출한다.
  서버 호출 실패 시 일부 기능은 로컬 fallback을 사용한다.

런타임 파일:
- Electron userData 경로에 license.json, runtime.json, settings.json, finance-category-memory.json 등이 저장된다.
- app\main.js가 `process.env.RUNTIME_PATH`를 설정하고, utils\runtimePaths.js가 이 경로를 기준으로 logs/browser profile/auth 폴더를 잡는다.

라이센스 흐름:
1. 고객이 앱에 서버 주소와 라이센스 키를 입력한다.
2. app\main.js가 기기 fingerprint를 만든다.
3. `/api/license/verify`로 서버 검증을 요청한다.
4. 서버가 라이센스 기능 권한과 만료/기기 바인딩 상태를 내려준다.
5. 고객 앱은 license.json에 integrityHash를 포함해 저장한다.
6. 실행 시 license.features에 따라 기능 접근을 제한한다.

기능 권한 키:
- baeminReply
- baeminReplyBasic
- baeminReplyPremium
- baeminBlind
- coupangReply
- coupangReplyBasic
- coupangReplyPremium
- coupangBlind
- naverMail
- financeAnalysis

답글 모드:
- 기본 답글: Basic. 로컬 별점/키워드 기반 답글 또는 GPT-mini 성격.
- 고급 답글: Premium. GPT 서버 중계 기반 답글.
- 현재 앱은 UI에서 답글 모드를 직접 선택하는 방식보다, 라이센스 기능 권한에 맞춰 자동 적용하는 방향으로 바뀌었다.

실행 제한:
- 한 번에 여러 자동화 기능을 동시에 선택하지 못하도록 바뀌었다.
- 실행 전 validateFeatureAccess, validatePreparedSettings 류의 검증을 거친다.

업데이트:
- app\main.js의 checkAndInstallUpdate가 `/api/updates/client/latest`를 호출한다.
- manifest의 version이 현재 app.getVersion보다 높으면 installer를 userData\updates 아래로 다운로드한다.
- 다운로드는 임시 파일 `.download`로 받고, 크기/sha256 검증 후 최종 exe로 교체하는 구조다.
- 기존 패치에서 다운로드 실패 원인을 “확인/다운로드/설치 실행” 단계로 나누도록 개선했다.

6. GPT 서버 중계 설계
============================================================

위치:
03_license_server\server.js

기술:
- Node.js 기본 http 서버
- 외부 프레임워크 없음
- OpenAI Responses API 형식으로 서버에서 fetch 호출

주요 서버 환경:
- HOST 기본값: env 기준
- PORT 기본값: env 기준
- ADMIN_SECRET: 관리자 앱 인증 헤더 `x-admin-secret`
- OPENAI_API_KEY: 서버에서만 보관
- OPENAI_MODEL_ADVANCED: 기본 고급 모델
- OPENAI_MODEL_BASIC: 기본 기본 모델

서버 주요 API:
- GET /health
  서버 상태 확인.

- GET 또는 HEAD /downloads/*
  업데이트 exe 다운로드.
  HEAD 요청과 Content-Length 응답을 지원하도록 보강됨.

- GET /api/updates/client/latest
  고객 앱 최신 버전 manifest 반환.

- GET /api/updates/admin/latest
  관리자 앱 최신 버전 manifest 반환.

- GET /api/admin/licenses
  관리자 라이센스 목록 조회.

- POST /api/admin/licenses
  관리자 라이센스 발급.

- POST /api/admin/licenses/toggle
  라이센스 활성/비활성 전환.

- POST /api/admin/licenses/extend
  라이센스 기간 연장.

- POST /api/admin/licenses/delete
  라이센스 실제 삭제.

- POST /api/license/verify
  고객 앱 라이센스 검증 및 최초 기기 바인딩.

- POST /api/gpt/reply
  리뷰 답글 생성.

- POST /api/gpt/review-care-apology
  배민/네이버 메일 게시중단 사과문 생성.

- POST /api/gpt/finance-classify
  거래내역 카테고리 분류.

- POST /api/gpt/store-click-analysis
  우리매장클릭 성과 분석.

- POST /api/gpt/threads-drafts
  스레드 글 초안 생성.

라이센스 데이터:
- 위치: 03_license_server\data\licenses.json
- 운영 서버 위치: /opt/chungdam-license-server/data/licenses.json
- 라이센스에는 licenseKey, customerName, issuedAt, expiresAt, isEnabled, deviceFingerprint, features, licenseHash 등이 포함된다.
- licenseHash는 변조 방지를 위해 주요 필드를 sha256으로 해시한다.
- 기존 구버전 hash와 호환하기 위한 legacy hash 검증도 있다.

기능 권한 정규화:
- normalizeFeatureMap에서 구형 `baeminReply`, `coupangReply`와 신형 Basic/Premium 권한을 함께 처리한다.
- 구형 라이센스에서 `baeminReply: true`만 있으면 명시 tier가 없을 때 고급 답글처럼 동작할 수 있다.

중요 안정화 포인트:
- 서버 manifest의 client version/fileName이 로컬 최신 산출물과 맞는지 확인해야 한다.
- `server.js` 기본 CLIENT_APP_VERSION이 `1.0.22`로 남아 있으므로, 운영 서버 env 또는 코드가 1.0.24로 맞춰졌는지 확인해야 한다.

   고객 PC에서 원인을 빠르게 회수하려면 user-errors JSONL, run log, 쿠팡 PNG/JSON 디버그 파일 경로가 화면에 잘 보여야 한다.

6. 인코딩 깨진 문서를 정리한다.
   패치내역_버전별.txt와 일부 README/사용법 문서가 깨져 보여 Cursor 작업자가 혼동할 수 있다.

이 프로젝트는 `C:\Users\DESKTOP\Desktop\코덱스`에 있는 청담 자동화 프로그램이다.
