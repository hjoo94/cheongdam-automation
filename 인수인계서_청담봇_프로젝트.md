## 5. 환경 변수 우선순위 (중요)

Node는 **이미 프로세스에 설정된 `process.env`가 있으면** `server.js`가 읽는 `.env` 로더가 덮어쓰지 않는 구현일 수 있음. 운영에서는 다음을 구분할 것.

1. **서버 셸 / systemd / pm2** 에 넘긴 환경 변수  
2. **`03_license_server/.env`**  
3. **`server.js` 내 폴백 문자열**

따라서 **`server.js` 폴백만 수정했다고 해서**, 서버에 **`CLIENT_APP_VERSION` / `CLIENT_APP_FILE` / `PUBLIC_BASE_URL` 이 `.env`에 박혀 있으면** 매니페스트는 계속 예전 값일 수 있음. 배포 후 **`Select-String` 또는 서버에서 `GET /api/updates/client/latest` 응답 JSON**으로 최종 확인.

## 6. 라이선스·자동 업데이트 (한 줄 요약)

- 라이선스: 클라이언트 → **`POST /api/license/verify`** → 서버 `data/licenses.json` 기준 검증.  
- 자동 업데이트: 클라이언트 → **`GET /api/updates/client/latest`** → `version` / `url` / `sha256` / `installerReady` → `downloads/` 아래 **실제 exe 파일명·해시**와 일치해야 사용자 PC에 패치 적용.

**배포 체크:** `03_license_server/downloads/`에 **`CLIENT_APP_FILE`과 동일한 파일명**의 설치 파일 업로드 → API에서 `installerReady: true` 및 `sha256` 64자 확인.

## 10. 모바일 웹 MVP (Claude 지시서 대비 실제 구현)

| 항목 | 내용 |
|------|------|
| 인증 | **JWT** — `POST /api/mobile/auth/login` (body: `licenseKey`, 선택 `deviceFingerprint`), 만료 **3600초**. `JWT_SECRET` 없으면 **서버 프로세스 시작 거부**. |
| 호스팅 | Lightsail **43.201.84.136:4300** 동일 베이스에서 **`GET /mobile/`** 으로 `05_mobile_mockup/mobile-dashboard-mockup.html` 서빙. |
| CORS | **`MOBILE_WEB_ORIGIN`** 이 있으면 해당 오리진만 `/api/mobile/*` 및 `/mobile` 응답에 CORS 헤더. 없으면 브라우저 **동일 오리진** 전제(파일 직접 열기 `file://` 는 API 호출 불가). |
| 로그인 속도 제한 | **`/api/mobile/auth/login`** IP당 **분당 10회**. |
| 이번 패치 제외 | UI 디자인 폴리싱, 쿠팡/배민 **자동화 플랫폼 코드** 수정 없음. |

**Claude 지시와 다른 점 (의도적)**

| 지시 | 실제 |
|------|------|
| `/api/mobile/*` 전부 Bearer 필수 | **`POST /api/mobile/state/update`**, **`POST /api/mobile/commands/pull`** 는 **기존대로 JSON body**(`licenseKey`+`deviceFingerprint`)만으로 인증 — **Electron 데스크톱 호환**. 나머지 `state/get`, `commands/create`, `commands/push`, `auth/login` 은 규칙에 맞게 분리. |
| `POST /api/mobile/commands/push` `{ command: "toggle" }` | 서버는 **`threads_emergency_stop` 명령을 토글**(상태에 따라 `enabled` 반전). **기능 토글 전체**가 아님. |
| `state/get` 를 GET만 가정 | **GET + POST** 모두 지원(본문 없이 Bearer만). |
| express-rate-limit | **미사용**. IP별 카운트 **직접 구현**. |
| 의존성 | **`jsonwebtoken`** — `03_license_server/package.json` 에 추가, **`npm install`** 필요. |

- **v1.0.31 구현 완료:** JWT 인증 + `mobile-dashboard-mockup.html` fetch 연동 + **`preview-mobile.html`** 미리보기 파일 추가.  
- **미리보기:** `05_mobile_mockup/preview-mobile.html` — 브라우저에서 **파일 직접 열기**(API 없음). 실제 연동은 **`/mobile/`** 또는 동일 호스트에서 대시보드 HTML 사용.

### v1.0.31 패치 요약·제한·에러 가능 지점

| 구분 | 내용 |
|------|------|
| 패치 요약 | `server.js` 업데이트 매니페스트 **1.0.31** / 클라이언트 `package.json` **1.0.31** / 모바일 대시보드 UI 교체·**실제 API 필드(`ok`, `state.status.threadsPolicy`)** 반영 / **preview-mobile.html** 추가. |
| Claude 지시서와 불일치 | 지시서는 **Express**(`app.post`, `res.json`) 예시 — 저장소는 **`http.createServer` 단일 파일** 구조 유지. JWT·rate limit·엔드포인트는 **이미 동등 구현**되어 있어 중복 삽입하지 않음. |
| 대시보드 응답 형식 | 서버는 `{ ok, token }`, `{ ok, state }`, `{ ok, id, threadsEmergencyStop }` — 지시서의 `{ success }` 와 **다름**. HTML은 **서버 실제 형식**에 맞춤. |
| `commands/push` | 서버는 **명령 큐에 `threads_emergency_stop` 삽입**; UI 배지는 **데스크톱이 `state/update`로 올린 스냅샷**을 읽음. 토글 직후 배지가 곧바로 바뀌지 않으면 **Electron이 pull·적용한 뒤** 다시 `state/update` 할 때까지 지연 가능. |
| 기기 바인딩 라이선스 | 로그인 시 **`deviceFingerprint`** 필수 — 대시보드에 선택 입력란 있음. |
