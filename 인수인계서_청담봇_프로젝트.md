**대상**: 이 폴더(`코덱스`)에 접근하는 다른 AI·개발자·운영 담당자  
**목적**: 저장소 전체 맥락, 빌드·배포, 서버·라이센스·클라이언트 연동을 한 문서에서 파악할 수 있게 함  
**작성 기준일**: 2026-04-14 (이후 버전·IP는 `package.json`, `.env`, 본 문서 상단 주석을 우선 확인할 것)

## 3. 고객 앱 `02_client_app/baemin-review-bot`

### 3.1 한 줄 요약

**Electron + Playwright**로 배민·쿠팡 리뷰 자동화(답글·저평점 케어), 네이버 메일 연동, 재무 분석, 스레드 마케팅 초안 등을 수행. **라이센스·GPT는 원칙적으로 라이센스 서버 경유**.

### 3.2 기술 스택

- Electron **28**, Node CommonJS  
- **Playwright** (Chromium 자동화)  
- **electron-builder** — Windows **NSIS** 설치 패키지 (`Cheongdam Bot Setup x.y.z.exe`)

### 3.3 엔트리·핵심 파일

| 파일 | 설명 |
|------|------|
| `package.json` | 앱 이름 `chungdam-bot`, **버전**, `build`·`asarUnpack` 정의. |
| `app/main.js` | 메인 프로세스: 창, IPC, **라이센스 검증**, 설정, **자동 업데이트**, 자식 러너(`runner.js`) spawn, 재무·스레드 IPC 등. |
| `app/preload.js` | `contextBridge`로 렌더러에 제한된 API 노출. |
| `app/index.html` | 고객 UI(설정, 기능 선택, 로그 등). |
| `app/security.js` | 서버 URL 정규화, **HTTP 허용 호스트(신뢰 IP)**, 설정 내 민감값 암호화 보조, **구 라이센스 서버 IP 자동 치환**. |
| `build/installer.nsh` | NSIS **커스텀** 조각: 설치 전 `Cheongdam Bot.exe` **트리 강제 종료** 후 진행(잠금 오류 완화). `package.json` → `nsis.include: "installer.nsh"`. |
| `index.js` | CLI/자식 프로세스 진입점. 플랫폼별 `run*` 호출. |
| `bot/runner.js` | 기능별 자동화 실행 루프(메인에서 base64 페이로드로 전달). |
| `bot/actions.js` | 기능 키 → `platforms/*.js` 모듈 매핑, 런타임 env 주입. |
| `gptClient.js` | `runtime.json` 기준으로 서버 `/api/gpt/*` 호출(일부 로컬 fallback). |
| `platforms/*.js` | 배민/쿠팡/네이버 메일 등 **사이트별 자동화 스크립트**. |
| `utils/*` | 로그, 오류 수집, 재무, 쿠팡 페이지네이션, 런타임 경로 등. |
| `scripts/smoke-tests.js` | Electron 없이 돌리는 **가벼운 회귀**(security URL 등). `npm test`. |

### 3.4 사용자 데이터 (Windows 예시)

Electron `userData` (일반적으로 `%AppData%\chungdam-bot\`):

- `settings.json` — 서버 URL, 매장·스레드 설정 등 (`threadsAccessToken` 등은 DPAPI 암호화 저장 시도).  
- `license.json` — 라이센스 키·검증 결과·`integrityHash`.  
- `runtime.json` — 메인이 동기화하는 런타임 스냅샷(자식·GPT가 읽음).  
- `updates/` — 자동 업데이트로 받은 설치 exe, **지연 설치용 `.cmd` 래퍼** 등.  
- `logs/` — 실행 로그, `user-errors-*.jsonl`, 쿠팡 해피톡 디버그 PNG/JSON 등.

### 3.5 라이센스·업데이트 흐름 (요약)

1. 사용자가 **서버 Base URL** + **라이센스 키** 입력 → `POST /api/license/verify` (기기 핑거프린트·무결성 리포트 포함 가능).  
2. 성공 시 `license.json` 저장, 기능 플래그로 UI·실행 제한.  
3. **자동 업데이트**: 여러 서버 후보에서 `GET /api/updates/client/latest` → 버전·sha256·url 검증 후 설치 exe 다운로드 → **Windows에서는 래퍼 cmd로 잠시 대기 후 taskkill·설치기 `/S`** 등으로 잠금 완화.  
4. **패키지 빌드에서 HTTP 라이센스 서버**를 쓰려면 `security.js`의 신뢰 호스트 + 환경변수 `CHUNGDAM_HTTP_TRUST_HOSTS` 등 규칙을 이해할 것.

### 3.6 빌드

```bash
cd 02_client_app/baemin-review-bot
npm install
npm test
npm run build
```

산출물: `dist/Cheongdam Bot Setup <버전>.exe`, `dist/win-unpacked/`.

**버전 올릴 때**: `package.json`만이 아니라 **`03_license_server`의 `CLIENT_APP_VERSION` / `CLIENT_APP_FILE`**, 서버 `downloads/`의 실제 파일명까지 **한 세트**로 맞출 것.

## 4. 라이센스 서버 `03_license_server`

### 4.1 한 줄 요약

**프레임워크 없는 `http.createServer` 하나**로 REST 비슷한 라우팅. OpenAI 키는 **서버에만** 두고 클라이언트는 `/api/gpt/*`로 중계.

### 4.2 주요 파일

| 파일 | 설명 |
|------|------|
| `server.js` | 전 라우팅·파일 응답·라이센스 CRUD·GPT fetch. |
| `.env` / `.env.example` | `PORT`, `HOST`, `ADMIN_SECRET`, `OPENAI_*`, **`PUBLIC_BASE_URL`**, **`CLIENT_APP_VERSION`**, **`CLIENT_APP_FILE`** 등. |
| `data/licenses.json` | 라이센스 DB(로컬 개발). 운영은 서버 동일 상대경로. |
| `data/mobile-state.json`, `data/mobile-commands.json` | 모바일 연동용(있다면). |
| `DEPLOY.md` | Lightsail Ubuntu 배포 절차. |
| `scripts/sync-client-installer.js` | (선택) 클라 설치 파일 동기화 보조 스크립트. |

### 4.3 API 요약 (경로만)

| 메서드·경로 | 용도 |
|-------------|------|
| `GET /health` | 헬스체크. |
| `GET /api/updates/client/latest` | 고객 앱 업데이트 매니페스트. |
| `GET /api/updates/admin/latest` | 라이센스 관리자 앱 매니페스트. |
| `GET/HEAD /downloads/*` | 설치 exe 등 정적 파일. |
| `GET/POST /api/admin/licenses` (+ toggle, extend, delete) | 관리자 전용, 헤더 `x-admin-secret`. |
| `POST /api/license/verify` | 고객 라이센스 검증·기기 바인딩. |
| `POST /api/gpt/reply`, `review-care-apology`, `finance-classify`, `store-click-analysis`, `threads-drafts` | GPT 중계. |
| `POST /api/mobile/state/update`, `.../get`, `.../commands/create`, `.../commands/pull` | 데스크톱↔모바일 상태(클라 `main.js`에서 사용). |

### 4.4 운영 배포 (요약)

- 일반 경로: **`/opt/chungdam-license-server`**, systemd 예: **`chungdam-license-server`**.  
- 방화벽: **TCP 4300**(및 SSH 22).  
- **공인 IP·`PUBLIC_BASE_URL`·클라이언트 기본 URL**이 어긋나면 라이센스·업데이트·다운로드 URL이 틀어짐.  
- 자세한 명령은 **`DEPLOY.md`**.

### 4.5 현재 운영 URL 메모 (변경 시 문서·코드 동시 수정)

- 라이센스 서버 기본(코드·문서 기준): **`http://43.203.124.132:4300`**  
- 구 IP **`43.201.84.136`** 은 레거시; 클라이언트 `security.js` 등에서 **자동 치환** 로직이 있음.

## 7. 다른 AI에게 줄 “최소 체크리스트”

새 작업을 시작할 때 아래를 **순서대로** 열어보면 맥락이 빠르게 잡힌다.

1. **`02_client_app/baemin-review-bot/package.json`** — 제품명·버전·NSIS·asarUnpack.  
2. **`02_client_app/baemin-review-bot/app/main.js`** (앞부분: 경로·서버 URL·업데이트·IPC 목록)  
3. **`02_client_app/baemin-review-bot/app/security.js`** — HTTP 신뢰·URL 정규화.  
4. **`03_license_server/server.js`** (상단 env·라우트 분기 부근)  
5. **`03_license_server/.env.example`** 또는 실제 `.env` 키 이름만(값은 노출 금지).  
6. **`03_license_server/DEPLOY.md`**  
7. **`04_license_admin/app/main.js`**  
8. **`.cursor/rules/chungdam-delivery.mdc`**  
9. (역사·세부) **`Cursor_인수인계_프로그램_설계_패치내역.txt`**

플랫폼 자동화만 수정할 때는 **`platforms/`** + **`bot/actions.js`** + 해당 기능의 **`utils/`**.

## 9. 알려진 운영 이슈 (패치 시 참고)

- **NSIS 업데이트**: 실행 중인 `Cheongdam Bot.exe`·자식 프로세스 때문에 “이전 버전 제거 실패” 또는 “앱을 닫을 수 없음” — `build/installer.nsh`, `main.js`의 `runInstaller` / `cleanupPreparedChildren`이 그 완화책.  
- **서버 IP 변경**: 클라이언트 기본 URL, `PUBLIC_BASE_URL`, 관리자 앱 기본값, `CHUNGDAM_HTTP_TRUST_HOSTS` 등 **한 번에** 점검.  
- **`node_modules`**: AI가 전체 grep 하면 노이즈가 큼 — 소스는 **`app/`**, **`platforms/`**, **`utils/`**, **`bot/`**, **`gptClient.js`**, **`index.js`** 우선.
