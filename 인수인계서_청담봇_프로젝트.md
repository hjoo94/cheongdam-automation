## 5. 환경 변수 우선순위 (중요)

Node는 **이미 프로세스에 설정된 `process.env`가 있으면** `server.js`가 읽는 `.env` 로더가 덮어쓰지 않는 구현일 수 있음. 운영에서는 다음을 구분할 것.

1. **서버 셸 / systemd / pm2** 에 넘긴 환경 변수  
2. **`03_license_server/.env`**  
3. **`server.js` 내 폴백 문자열**

따라서 **`server.js` 폴백만 수정했다고 해서**, 서버에 **`CLIENT_APP_VERSION` / `CLIENT_APP_FILE` / `PUBLIC_BASE_URL` 이 `.env`에 박혀 있으면** 매니페스트는 계속 예전 값일 수 있음. 배포 후 **`Select-String` 또는 서버에서 `GET /api/updates/client/latest` 응답 JSON**으로 최종 확인.

## 6. 라이선스·자동 업데이트 (한 줄 요약)

- 라이선스: 클라이언트 → **`POST /api/license/verify`** → 서버 `data/licenses.json` 기준 검증.  
- 자동 업데이트: 클라이언트 → **`GET /api/updates/client/latest`** → `version` / `url` / `sha256` / `installerReady` → 서버 **`downloads/`** (실제 구현은 **`03_license_server/downloads/`** 등 `server.js`가 읽는 경로) 아래 **실제 exe 파일명·해시**와 일치해야 사용자 PC에 패치 적용.

**배포 체크:** 운영 서버의 downloads 디렉터리에 **`CLIENT_APP_FILE`과 동일한 파일명**의 설치 파일 업로드 → API에서 `installerReady: true` 및 `sha256` 64자 확인.

### 12.2 Git·워킹트리에서 주의한 점

- **`git add -u` 일괄 스테이징 주의:** 로컬에서 **`인수인계서_청담봇_프로젝트.md`**, **`클로드_컨텍스트_복원_메모.md`** 가 삭제된 상태였다면, 무분별 `-u`로 **중요 문서까지 삭제 커밋**될 수 있음. 해당 경우 **`git restore <파일>`** 로 복구한 뒤, **삭제 대상 파일만** 스테이징할 것.  
- **미추적 파일:** 예) `패치파일_EXE_여기/Cheongdam Bot Setup 1.0.25.Zip` — 필요 시 `.gitignore` 또는 팀 규칙으로 정리. **실수로 `git add .`에 포함되지 않게** 주의.

### 12.3 빌드 에러 (electron-builder)

- **증상:** `npm run build` 중 `remove ... dist\win-unpacked\Cheongdam Bot.exe: The process cannot access the file because it is being used by another process`  
- **원인:** 이전 빌드 산출물·실행 중인 Electron/청담봇·탐색기 등이 **`win-unpacked` 내 exe를 점유**.  
- **대응:** (1) 관련 프로세스 종료 후 `dist\win-unpacked` 또는 전체 `dist` 삭제 → `npm run build` 재실행. (2) 잠금이 계속되면 **출력 폴더만 분리:**  
  `npx electron-builder --win --x64 --config.directories.output=dist1032`  
  → 산출물 예: `dist1032\Cheongdam Bot Setup 1.0.32.exe`
